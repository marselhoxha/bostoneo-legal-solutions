Context

What changed, what didn't

The wizard atoms (stepper, practice-area grid, document-type cards, jurisdiction selector, LOR radios, template-resolution card) already match the original preview.html. Sprint 4a fixes the frame around the wizard — the page-level layout, arrival behavior, and Case Context — plus four small in-wizard fidelity misses.

IN SCOPE
· Dedicated 2-column focused layout when user clicks LegiDraft
· Empty state with "+ New Draft" and "Use Template" CTAs
· Right-rail Case Context panel (always visible)
· Close button (×) to exit draft mode
· Wizard state resets on each new draft
· LOR: add missing 4th recipient option
· LOR: verify 5 purposes + recipient-conditional enable/disable
· LOR: info alert + client-insurance empty warning
· Step 4: right-column Summary card
· Post-wizard generation view with workflow pills
OUT OF SCOPE
· New Angular route (decision: keep view-mode in ai-workspace)
· Recent Drafts left sidebar (deferred to Sprint 4b)
· Backend changes (everything already in place from Sprint 1)
· Rewriting the wizard atoms (they already match)

· New Flyway migration
· Mobile/tablet responsive pass (deferred to Sprint 4b)

Frame G

Exit flows — two buttons, two intents

Cancel and Close serve different purposes. Mixing them is a common UX bug; separating them prevents accidental loss of intent.

"Cancel" on any wizard step
Cancel the current draft
Returns to the empty state (stays in draft mode)
TriggerCancel button inside the wizard
EffectClears wizard fields, showDraftWizard = false
Lands onEmpty state with 2 CTAs (Frame B)
BreadcrumbLegiSpace › LegiDraft
Use whenAttorney wants to redo the draft but keep exploring LegiDraft
"×" Close button (top-right)
Exit draft mode entirely
Returns to the 4-card LegiSpace welcome
Trigger× button top-right of focused layout
EffectselectedTask = null, wizard state cleared
Lands on4-card LegiSpace welcome (Frame A)
BreadcrumbLegiSpace › Welcome
Use whenAttorney is switching to a different LegiTool (search, lyze)
Common mistake we're avoiding. If Cancel and × did the same thing, a wizard-weary attorney who clicks Cancel would lose the entire LegiDraft context — unhelpful. The split lets them "try again" without teleporting out.
Frame H

Dark mode — tokens flip automatically

No hardcoded colors inside .legidraft-focused or any child component. Velzon's [data-bs-theme="dark"] on <html> flips every token; the skin only overrides font-family.
