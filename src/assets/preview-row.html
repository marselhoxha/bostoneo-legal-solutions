List view · row redesign

How do we surface progression, subtasks, time logged in-row?

Currently the row shows title · case · priority · due · assignee · ⋯. Subtasks count, time logged, and overall progression live only in the drawer (one click away). For attorneys triaging a queue, a tiny extra signal in the row would let them prioritize without opening every task. Three directions below differ on a single axis: where the new info goes and how dense it gets.

  • D1 Progression Bar — same row layout, add a thin 2px bar at the bottom of each row showing % subtask completion. Lightweight, scannable.
  • D2 Two-Line Stack — title on top, full metadata line below (case · priority · due · subtasks · time · assignee). Each row is taller; clearer hierarchy at the cost of density.
  • D3 Inline Indicators — same row layout, add small chip-style indicators between title and case-pill: ⊞ subtasks · ⏱ time. Information-rich without breaking the row height.

All three use the same realistic data so you can compare honestly: 6 representative tasks across Hoxha, Sullivan, Chen, Wright, Meridian, Dorsey cases with varying subtask completion (0/0, 2/4, 5/5), time logged (0h, 1.4h, 8.5h), priority levels, and due states.

Direction 1

Progression Bar — thin progress strip on each row

Same column layout as today. A 2px progress bar runs along the bottom edge of each row showing % subtask completion (or, for tasks without subtasks, % of estimated hours logged). Color: blue for in-progress, green for complete, red for overdue+stalled. Subtask count and time logged appear as faint inline metadata after the title (e.g., "Review Hoxha records · 2/4 · 1.4h").

Best for: attorneys who want subtle signal · keeps row density at current 51px height · weakest at large counts (you can't read "2/4" mid-scan if there are dozens of rows)

Direction 2

Two-Line Stack — title on top, metadata below

Each row becomes ~64px tall (vs 51px). Title sits on the first line. Below it, a single horizontal metadata strip shows: case (with color dot) · subtasks completion (with mini progress bar) · time logged · due date. Priority pill + assignee + ⋯ menu stay at the right edge. Reads like a "card list" without becoming a card. Best when scrolling for context, weakest at fitting many rows in one viewport.

Best for: review / triage sessions · pairs well with Compact toggle that collapses to D1-style on demand · heaviest visual change of the three

Direction 3

Inline Indicators — chip-style metrics in the row

Same row height as today (51px). Adds two small chip-style indicators between the title and the case pill: ⊞ 2/4 for subtasks completion and ⏱ 2.4h for time logged. Indicators turn green when subtasks complete (3/3 done), warning when time exceeds estimate, danger when blocked. Information-rich without breaking density. Easiest to scan because every metric has a dedicated visual location.

Best for: power users with large queues · keeps row density · adds 2 columns to the grid · fewest tradeoffs
